## Key finding: neither the city subtotal nor the grand total was printed

The table has a "Total" row after the 18 city precincts and a "Grand total"
row after all ~68 precinct rows — **both are printed completely blank** (dots
only, no digits), in every column, confirmed at
`mirror/cdnc/screenshots/sd_18981109_p2_totalrow.png` (city Total) and
`mirror/cdnc/screenshots/sd_18981109_p2_crop_grandtotal_full.png` /
`sd_18981109_p2_seg7.png` (Grand total). The compositor ran out of time
before 4:30am press (consistent with the narrative) and never filled in the
summary rows. There is no shortcut: the floor here can only be built by
summing every individual precinct row by hand.

## Method note: table legibility and a caught transcription error

This table's ~35 columns are compressed into roughly half the page width, so
each cell is only a few pixels wide even after 2x crop-upscaling. A
combined single-crop technique (precinct-label column + Gage/Neff/Maguire/
Hutchinson columns cropped together in one image, so rows stay aligned by
construction) worked far better than separately cropping labels and numbers
(which caused a row-count mismatch on the first attempt). Even so, a
**row-attribution error was caught and corrected** during this pass: an
early read attributed the Sixth Ward-1st Pre. figures (Gage 107, Maguire 95)
to Fifth Ward-2d Pre.; a follow-up tight re-crop
(`mirror/cdnc/screenshots/sd_18981109_p2_sixth1.png`) fixed it. Given one
error was caught by spot-checking, others likely remain uncaught in a table
this dense — **confidence for this entire table is LOW**, distinct from the
1892 dossier's per-digit HIGH confidence. A same-session attempt to fetch
the issue's higher-resolution static PDF via
`/Users/sbuss/.claude/jobs/8f24b289/tmp/sd/cdnc_pdf.js` (for a cleaner
re-read) did not complete (180s timeout, no file produced); a future pass
should retry that or pull the microfilm-resolution image before this number
is treated as reliable.

columns** (no returns received or not yet phoned in by 4:30am press,
